## Recent main-branch fixes (all merged)
- Polling tooltip: stripped Tailwind visibility classes, owns visibility/fade/position-flip via inline `<style>` block in createElements(). Class `polling-group` added to parent for `:hover` selector that doesn't depend on Tailwind escaping. Padding-based hover bridge: `padding-top: 12px` (maximized) and `padding-bottom: 12px !important` (minimized) so cursor stays in tooltip's hover region while crossing the 12px gap to the slider.
- Minimized panel corners: `dashboard-header` class on the header div with `border-top-left-radius: 24px; border-top-right-radius: 24px;` in inline style block. Solves the conflict where `overflow: visible` on the minimized panel (needed to let tooltip escape) caused the inner header background to paint over the panel's rounded corners.
- Tab drag-and-drop reorder: native HTML5 DnD, no library. Lifted look via opacity 0.4 + scale 1.02. Slot highlight via 3px emerald bar from ::before/::after pseudo-elements. New `reorderTab(fromIdx, toIdx)` method on tab controller persists via existing save() path.
- Polling tooltip clipping fix: applyMinimizedStyles() sets `panel.style.overflow = 'visible'`; applyMaximizedStyles() restores `'hidden'`.

## Communication notes
- Hardware mode = design/planning, no code
- Software mode = code generation OK
- User holds strong MVC opinions, calls out architectural violations sharply
- "Dev is where the work gets done; main is the stable baseline."
- DO NOT ask for files from the repo — fetch directly using the URL pattern
- DO NOT pull `main.js` — production bootstrap is `scripts/neonova-dashboard.user.js`
- URL fetcher constraint: only accepts URLs the user has explicitly pasted in conversation OR that appeared in search results. If a URL fetch fails, ask user to paste the raw URL.
- Long conversations degrade assistant accuracy — start fresh between phases.
- "Blame the cache" is a lazy diagnosis. Do not reach for it as first guess.
- When user says "give me the methods" prefer separate code blocks per method, not one big dump.
- When user is on dev/main/demo, REMEMBER which. Tailwind class availability differs:
  - Demo: precompiled tailwind.css with explicit safelist. Classes not in safelist produce no CSS.
  - Dev/main: runs as userscript on host page (admin.neonova.net). Tailwind from host or none.
- When making CSS changes that depend on Tailwind utilities, prefer owning rules in inline `<style>` blocks rather than relying on utility classes that may or may not exist.
